## Auto-Generated Documentation

These files are automatically generated by the system:

### `.copilot/docs/discovery.json`
Project metadata including:
- Total files count
- Tech stack detection
- File categorization
- Entry points

### `.copilot/docs/architecture/overview.md`
Architecture overview including:
- Project structure
- Key directories
- Technology stack
- Coding patterns

### `.copilot/docs/codemaps/*.json`
File categorization maps:
- `frontend-map.json` - Frontend files
- `backend-map.json` - Backend files
- `shared-map.json` - Shared utilities
- `test-map.json` - Test files

### `.copilot/docs/maven-discovery.json` (Maven projects)
Maven-specific metadata:
- Module structure
- POM information
- Java file categorization

### `.copilot/docs/spring-xml-discovery.json` (Spring XML projects)
Spring XML-specific metadata:
- Spring XML configs
- Bean definitions
- Component scan packages
- Internal libraries

### `.github/copilot-instructions.md`
Context for GitHub Copilot including:
- Project overview
- Coding standards
- Architecture patterns
- Common patterns

## Commands Reference

### From Project Root

```bash
# Update context (most used)
npm run copilot:update

# Watch mode (auto-update)
npm run copilot:watch

# Full regenerate
npm run copilot:init
```

### From .copilot/ Directory

```bash
# Update context
npm run update

# Run discovery
npm run discover

# Generate docs
npm run generate

# Run tests
npm test

# Maven discovery (if applicable)
npm run discover:maven

# Spring XML discovery (if applicable)
npm run discover:spring
```

## Documentation Maintenance

### When to Update Documentation

**User documentation (manual):**
- When adding new features
- When changing workflow
- When fixing bugs that affect usage

**Auto-generated documentation:**
- Automatically updated by `npm run copilot:update`
- Update 2-3 times per day
- After significant code changes

### How to Update

**User documentation:**
1. Edit markdown files directly
2. Ensure cross-references are correct
3. Update CHANGELOG.md
4. Commit changes

**Auto-generated documentation:**
1. Run `npm run copilot:update`
2. Review generated files
3. Commit if needed (optional)

## Documentation Standards

### File Naming
- `UPPERCASE.md` - User documentation
- `lowercase.md` - Auto-generated documentation
- `kebab-case.json` - Data files

### Cross-References
- Always use relative paths: `[Link](./FILE.md)`
- Include section anchors: `[Link](./FILE.md#section)`
- Keep links up to date

### Language
- User guides: Vietnamese (HUONG_DAN.md, MAVEN_GUIDE.md, SPRING_XML_GUIDE.md)
- Technical docs: English (README.md, QUICKSTART.md)
- Code: English (comments, variables)
